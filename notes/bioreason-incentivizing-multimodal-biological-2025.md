## TL;DR

BioReason is the first architecture that deeply integrates a frozen DNA foundation model (Evo2-1B or NT-500M) with an LLM (Qwen3-1.7B/4B) via a learned linear projection, enabling the LLM to reason over genomic embeddings alongside text. Trained with SFT (LoRA) then GRPO reinforcement learning, it produces step-by-step mechanistic reasoning traces. On a novel KEGG-derived disease-pathway benchmark it reaches 98.28% accuracy (vs 86–90% single-modality baselines) and delivers ~15% average gains on variant effect prediction tasks.

## Model

- **Architecture**: Frozen DNA encoder → learned linear projection (d_dna → d_llm) → concatenated with text embeddings via special `<dna_start>`/`<dna_end>` tokens → autoregressive LLM backbone.
- **DNA encoders tested**: Evo2-1B (Striped Hyena 2), Nucleotide Transformer 500M (NT-500M), Enformer (supervised, appendix experiment).
- **LLM backbone**: Qwen3-1.7B and Qwen3-4B.
- **Trainable components**: LLM weights (via LoRA), linear projection layer, special token embeddings. DNA encoder weights are frozen.
- **Reasoning format**: Output Y_OUT structured with `<think>…</think>` reasoning traces followed by a final answer.
- **Inference**: Deterministic decoding (temperature 0), served via vLLM.

## Data

- **KEGG-Derived Biological Reasoning Dataset** (novel): 1,449 entries linking genetic variants to 37 unique diseases via mechanistic pathway reasoning. Integrates KEGG pathway networks with variant data from ClinVar, dbSNP, OMIM, COSMIC. Paired ref/variant DNA sequences (~4,000 bp avg). Reasoning paths (mean 303.8 words) generated by Claude 3.7 Sonnet grounded on KEGG disease descriptions. 10% of train used for validation.
- **VEP-Coding**: 50,083 core entries from GPN-MSA / ClinVar / gnomAD. Train: Chr 1–7, 9–22, X, Y; Test: Chr 8. Augmented with 50 GPT-4o question paraphrases per sample.
- **VEP-Non-SNV**: 36,088 coding non-SNV entries from ClinVar (≤64 bp, ≥2-star review). Stratified train/test splits. Same 50× GPT-4o augmentation.

## Training Recipe

1. **Supervised Fine-Tuning (SFT)**: AdamW, lr 5e-5, weight decay 1e-2, gradient accumulation 8 steps. LoRA rank 32, alpha 64, dropout 0.05. KEGG task: 5 epochs, batch 1; VEP tasks: 3 epochs, batch 2. Max DNA length 2,048 tokens (truncated); max text length 1,024 (LLM-only uses 8,192 to fit raw DNA strings). Attention mask restricts loss to response tokens only (between `<think>` and final answer).
2. **Reinforcement Learning (GRPO / Dr. GRPO)**: Applied to select SFT-trained models. AdamW, lr 1e-5, cosine schedule with 0.03 warmup. LoRA rank 16, alpha 32, dropout 0. 8 generations per query, 1,000 steps (~7 epochs), 2 GPUs (4× A100 for RL). Composite reward (max 2.5): correctness +2.0 (case-insensitive substring match), conciseness +0.5 (≤10 words in final answer), format +0.5 (valid `<think>` block). ε=0.2, β=0.0 (no KL penalty). Temperature 0.7 (4B) / 1.0 (1.7B), top-p 0.95, top-k 20.
3. **Hardware**: NVIDIA A100 and H100 GPUs. DeepSpeed Stage 2. Single H100 for SFT; 4× A100 for GRPO.

**Key takeaways**: (1) Multimodal DNA-LLM consistently beats both single-modality baselines. (2) GRPO dramatically helps larger (4B) backbones but hurts F1 of smaller (1.7B) ones — model capacity is a prerequisite for effective RL. (3) Choice of DNA encoder (Evo2 vs NT) matters less than LLM backbone size. (4) Architecture generalizes to supervised DNA encoders (Enformer).

## Reported Insights

- LLMs treating DNA as plain text strings miss rich genomic features; DNA FMs capture them but produce opaque embeddings — the multimodal bridge solves both.
- The `<think>` token-structured reasoning traces provide interpretable, step-by-step mechanistic explanations (e.g., 10-step trace from PFN1 variant → actin dynamics disruption → motor neuron degeneration → ALS).
- GRPO is architecture-agnostic w.r.t. the DNA encoder: NT-based and Evo2-based models paired with Qwen3-4B follow nearly identical learning curves.
- DNA sequences truncated to 2,048 tokens due to hardware constraints — potentially omitting distal regulatory context.
- No uncertainty quantification; single-run evaluations (deterministic decoding, no error bars across seeds).

## References Worth Chasing

- **Evo2** [6] (Brixi et al. 2025): Largest genomic FM, Striped Hyena 2 architecture, long-range context — used as primary DNA encoder.
- **ChatNT** [35] (Richard et al. 2024): Prior multimodal DNA-LLM integration — but not trained for complex reasoning.
- **Dr. GRPO** [27] (Liu et al. 2025): Unbiased GRPO variant improving token efficiency — the specific RL algorithm used here.
- **DeepSeek-R1** [12]: Inspiration for incentivizing reasoning via RL in LLMs.

- **rbio1** [18] (Istrate et al. 2025): Scientific reasoning LLM with biological world models as soft verifiers — concurrent related work.

## Notes / Open Questions

- No error bars or multi-seed experiments reported; all results are single-run with deterministic decoding. Reproducibility/variance unknown.
- Reasoning traces are generated by Claude 3.7 Sonnet during data curation — how much of BioReason's reasoning ability is distilled from Claude vs learned de novo?
- GRPO β=0 means no KL penalty to the reference policy — aggressive but apparently stable for 4B; may explain instability for 1.7B.
- The 50× GPT-4o question paraphrasing for VEP tasks is a significant data augmentation — unclear how much performance depends on this.
- DNA truncation at 2,048 tokens limits applicability to long-range regulatory elements and whole-genome analysis.
- Code and checkpoints at https://github.com/bowang-lab/BioReason — worth checking actual model weights and data pipeline.
